A better SublimeREPL for Chez Scheme REPL and practice

If neither of the following 4 applications can satisfy you in Chez Scheme learning, you could turn to SublimeREPL.

  1. Terminal. Chez Scheme contains a REPL environment, but too many parentheses have to be keyed in it in Terminal. Moreover, you cannot correct typos conveniently in Terminal.
  2. Vim. After setting up the .vimrc, you just have to type only one parenthesis every time, but its syntactic support for Chez Scheme is bad such as autoindent does not work for Chez Scheme and you have to quit it to Terminal if you want to run a code snippet. So much unnecessary finger dancing.
  3. DrRacket. Although it is rebuilt on Chez Scheme, it does not support Chez Scheme well and even Scheme(R6RS) itself. You will come across some strange error warnings. You can try some code snippets of The Scheme Programming Language(4th edition), especially the chapter 2.
  4. VScode. No taste? The GUI is too complicated. Its REPL function works terribly even though it offers a Chez Scheme REPL plug-in. The most important thing is that it is as RAM memory-hogging as DrRacket. Both of their user experience is bad.

As to SublimeREPL (for MacOs, Linux does likewise), you have just to make three changes for a better experience.

  1. In ‘Sublime Text/Packages/SublimeREPL/config/Scheme’ (via ‘settings-browse packages’ of Sublime), under ‘“id”: “repl_scheme”‘:
    change:
    "osx": ["scheme"],
    to:
    "osx": ["scheme", "$file"],
  2. In the same place as #1. :
    change:
    "syntax": "Packages/sublime-scheme-syntax/Scheme.tmLanguage"
    to:
    "syntax": "Scheme.sublime-syntax" (to get this file, please refer to this post or download here)
    So you can avoid the annoying pop-up of ‘error loading loading syntax file: “Scheme.tmLanguage”‘. Or you can just comment this line.
