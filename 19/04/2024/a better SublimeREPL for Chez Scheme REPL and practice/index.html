a better SublimeREPL for Chez Scheme REPL and practice
19 April 2024

If none of the following 4 applications can satisfy you in Chez Scheme learning, you should turn to SublimeREPL.

  1. Terminal. Chez Scheme contains a REPL environment, but too many parentheses have to be keyed in it in Terminal. Moreover, you cannot correct typos conveniently in Terminal.
  2. Vim. After setting up the .vimrc, you just have to type only one parenthesis every time, but its syntactic support for Chez Scheme is bad such as autoindent does not work for Chez Scheme and you have to quit it to Terminal if you want to run a code snippet. So much unnecessary finger dancing.
  3. DrRacket. Although it is rebuilt on Chez Scheme, it does not support Chez Scheme well and even Scheme(R6RS) itself. You will come across some strange error warnings. You can try some code snippets of The Scheme Programming Language(4th edition), especially the chapter 2.
  4. VScode. No taste? The GUI is too complicated. Its REPL function works terribly even though it offers a Chez Scheme REPL plug-in. The most important thing is that it is as RAM memory-hogging as DrRacket. Both of their user experience is bad.

As to SublimeREPL (for MacOs, Linux does likewise), you have just to make three changes for a better experience.

  1. In ‘Sublime Text/Packages/SublimeREPL/config/Scheme’ (via ‘settings-browse packages’ of Sublime), under ‘“id”: “repl_scheme”‘:
    change:
    "osx": ["scheme"],
    to:
    "osx": ["scheme", "$file"],

  2. In the same place as #1. :
    change:
    "syntax": "Packages/sublime-scheme-syntax/Scheme.tmLanguage"
    to:
    "syntax": "Scheme.sublime-syntax" (to get this file, please refer to this post or download here)

    or, if you have installed the package of ‘Scheme’ with package control (command+shift+p), to:
    "syntax": "Packages/Scheme/syntax/Scheme.tmLanguage”, because the path of the syntax file is ‘Scheme.sublime-package/syntax/Scheme.sublime-syntax’ if you unzip the package.
    So you can avoid the annoying pop-up of ‘error loading syntax file: “Scheme.tmLanguage”‘. Or you can just comment this line.

    or, if you do not want to change the codes, just download the file sublime-scheme-syntax here and put it into the folder ‘Applications/Sublime Text/Contents/MacOs/Packages’ (MacOs. For Linux, perhaps it may be ‘/opt/sublime_text/Packages’ if you follow the official installation instructions.). We do not know why Sublime Text 4 does not include this file, while a ‘Lisp.sublime-package’ is in the same folder which also works for Scheme syntax if you just change ‘Lisp’ to ‘Scheme’.

  3. To run your codes and call the REPL in just one keystroke, add below lines in ‘Key Bindings’ (via ‘settings’): 

    {
  "keys": ["command+r"],
  "caption": "SublimeREPL: Scheme",
  "command": "run_existing_window_command",
  "args": {
  "id": "repl_scheme",
  "file": "config/Scheme/Main.sublime-menu"
   }
}

    In addition, to get a perfect indentation in SublimeREPL, you also have to configure the ‘repl_view_settings’ of ‘SublimeREPL.sublime-settings’ under ‘repls’ folder of Package SublimeREPL as below:

    "repl_view_settings": {
    "translate_tabs_to_spaces": true,
    "auto_indent": true,
    "smart_indent": true,
    "spell_check": false,
    "indent_subsequent_lines": true,
    "detect_indentation": true,
    "auto_complete": true,
    "line_numbers": false,
    "gutter": false
},

    Then let’s code happily with Chez Scheme.
